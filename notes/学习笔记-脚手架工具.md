#### Yeoman

使用

1. 在全局范围内安装 yo

```shell
npm install yo --global # or yarn global add yo
```

2. 安装对应的 generator

```shell
npm install generator-node --global # or yarn global add generator-node
```

3. 通过 yo 运行 generator

```shell
cd path/to/project-dir
mkdir my-module
yo node
```



##### sub generator

```shell
# 安装对应子集
yo node:cli

# mac 下 chmod +x lib/cli.js
yarn link
my-module --help
```

##### Plop

小而美的脚手架工具

###### 基本使用

- 将 plop 模块作为项目开发依赖安装
- 在项目根目录下创建一个 plopfile.js 文件
- 在 plopfile.js 文件中定义脚手架任务
- 编写用于生成特定类型文件的模板
- 通过 Plop 提供的 CLI 运行脚手架任务



#### 脚手架工作原理

通过命令行交互询问用户问题

根据用户回答的结果生成文件

### 自动化构建



#### 简介

源代码 => 自动化构建 => 生产代码



自动化构建工作流

脱离运行环境兼容带来的问题

使用提高效率的语法、规范和标准



- ECMAScript Next
- Sass
- 模板引擎

上述技术大都不被浏览器直接支持，使用自动化构建工具，构建转换那些不被支持的特性



案例：

开发人员

↓编写

```css
/* style.css */
body {
  margin: 0 auto;
  padding: 20px;
  max-width: 800px;
  background-color: #f8f9fb;
  color: #333;
}
```

↓运行

浏览器



在编写样式环节，希望通过编写 sass 增强 css 可编程性，再通过 sass 构建 css

```scss
// main.scss
$body-bg: #f8f9fb
$body-color: #333

body {
	margin: 0 auto;
	padding: 20px;
	max-width: 800px;
	background-color: $body-bg;
	color: $body-color;
}
```



简易 npm scripts 工作流

```js
{
  "scripts": {
    "build": "sass scss/main.scss css/style.css --watch", // 监听 sass 文件变化，实时编译
    "serve": "browser-sync . --files \"css/*.css\"", // 开启调试网页，监听文件变化，刷新
    "start": "run-p build serve" // 执行编译 sass，开启调试网页
  }
}
```

###### Grunt 具体使用

grunt.registerTask

grunt.initConfig({...}) // 配置选项方法

// 多目标任务



// 插件使用

yarn add grunt-contrib-clean

1. npm 安装
2. grunt.loadNpmTasks('grunt-contrib-clean') // 将插件中提供的任务加载进来
3. grunt.initConfig 中添加配置选项

```js
grunt.initConfig({
  clean: {
    temp: 'temp/app.js' // 'temp/*.txt' // 'temp/**' 
  }
})
```



常用的插件

- grunt-sass

yarn add grunt-sass sass --dev

```js
const sass = require('sass')

module.exports = grunt => {
grunt.initConfig({
  sass: {
    options: {
      sourceMap: true,
      implementation: sass
    },
    main: {
      files: {
        'dist/css/main.css': 'src/scss/main.scss'
      }
    }
  }
})

grunt.loadNpmTasks('grunt-sass')
  
}
```



- grunt-babel

yarn add grunt-babel @babel/core @babel/preset-env --dev

- load-grunt-tasks

yarn add load-grunt-tasks --dev

```js
const sass = require('sass')
const loadGruntTasks = require('load-grunt-tasks')

module.exports = grunt => {
  grunt.initConfig({
    sass: {
      options: {
        sourceMap: true,
        implementation: sass
      },
      main: {
        files: {
          'dist/css/main.css': 'src/scss/main.scss'
        }
      }
    },
    babel: {
      options: {
        sourceMap: true,
        presets: ['@babel/preset-env']
      },
      main: {
        files: {
          'dist/js/app.js': 'src/js/app.js'
        }
      }
    },
    watch: { // yarn grunt watch
      js: {
        files: ['src/js/app.js'], // ['app/js/*.js']
      	tasks: ['babel']
      },
      css: {
        files: ['src/scss/*.scss'],
        tasks: ['sass']
      }
    }
  })
  
  // grunt.loadNpmTasks('grunt-sass')
  loadGruntTasks(grunt) // 自动加载所有的 grunt 插件中的任务
	
  grunt.registerTask('default', ['sass', 'babel', 'watch'])
}
```
